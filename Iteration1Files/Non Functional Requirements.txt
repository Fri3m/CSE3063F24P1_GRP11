Performance: The system must respond to user inputs promptly, even as the number of users and data increases.

Scalability: The system should be able to handle an increasing number of courses, students, and advisors without a significant decrease in performance.

Reliability: The system must be reliable, minimizing downtime and ensuring data integrity.

Maintainability: The codebase should be well-organized and documented to facilitate maintenance and future updates.

Repository Size: The size of the GitHub repository, including all artifacts, should not exceed 10 MB.

Project Timeline: Each iteration must be completed within the designated three-week time frame.

Team Collaboration: Team members should use GitHub for collaboration, managing issues, a kanban board, and individual contributions on this platform.
